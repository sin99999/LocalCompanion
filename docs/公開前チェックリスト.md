# 公開前チェックリスト

公開 ZIP を配布する前に、毎回同じ手順で確認します。
所要時間の目安は 30〜45 分です（モデルダウンロード時間を除く）。

## 0. 事前準備

- [ ] `CHANGELOG.md` に今回の変更を記載した
- [ ] `LocalCompanion.csproj` の `<Version>` を上げた（例: 1.0.0 → 1.0.1）
- [ ] `git status` がクリーン（コミット済み）である

## 1. ビルドと ZIP 作成

- [ ] `dotnet build LocalCompanion.csproj -c Release` が警告なしで成功する
- [ ] `.\scripts\package-user-zip.ps1` が成功し、`dist\` に ZIP ができる
- [ ] ZIP 名にバージョンが入っている（例: `LocalCompanion-1.0.0-user.zip`）
- [ ] ZIP 直下に `THIRD-PARTY-NOTICES.txt`・`CHANGELOG.md`・`docs\Troubleshooting.md` がある
- [ ] `appsettings.json` に絶対パスが含まれていない（スクリプトが検証）

## 2. クリーン環境での起動確認

別フォルダー（できれば別 PC または新規ユーザー）に ZIP を展開して確認します。

- [ ] `LocalCompanion.exe` ダブルクリックで起動する
- [ ] 初回: llama.cpp・既定モデルの自動ダウンロードが完了する
- [ ] ⚙ **設定 → モデル** の実行状態が「OK」になる

## 3. 機能スモークテスト

- [ ] チャット: 短い質問に返答が返る（ストリーミング表示）
- [ ] チャット: 生成中の停止ボタンが効く
- [ ] 会話履歴: 保存され、左ペインから再開できる
- [ ] RAG: テキストファイルを 1 件登録し、内容について質問して回答に反映される
- [ ] 添付: テキストファイル添付・URL 読み込みが動く
- [ ] 画像: （mmproj 対応モデル時）画像添付で内容を説明できる
- [ ] VOICEVOX: インストール済み環境で読み上げが動く／未インストール環境で他機能に影響がない

## 4. 設定画面

- [ ] 全タブ（基本・モデル・キャラクター・RAG・VOICEVOX）が開ける
- [ ] バージョン表示が今回のバージョンと一致する
- [ ] ライセンス情報（THIRD-PARTY-NOTICES）が開ける
- [ ] ログフォルダーが開ける
- [ ] データのバックアップ（ZIP 書き出し)が動く
- [ ] 言語を English に切り替えて主要画面の文言が切り替わる → 日本語に戻す

## 5. 終了とデータ

- [ ] アプリ終了で llama-server・（本セッションで起動した）VOICEVOX が止まる
- [ ] 配布版: `{exe}\data\` にデータが作られている（`%LocalAppData%` ではない）
- [ ] フォルダーごと別の場所へ移動しても起動・データ引き継ぎができる

## 6. 配布物の最終確認

- [ ] ZIP 内に個人用 GGUF・キャラクター JSON・会話データが含まれていない
- [ ] `models\` `characters\` には README のみが含まれている
- [ ] README の手順どおりに第三者がセットアップできる内容になっている

---

問題が見つかった場合は修正後、関係する項目のみ再確認します。
バージョンを上げた場合は 1. からやり直します。
