# トラブルシューティング

LocalCompanion が動かない・調子が悪いときは、まずこのページをご確認ください。

## 起動しない

### exe をダブルクリックしても何も起きない／すぐ閉じる

**.NET 10 Desktop Runtime（x64）が未インストールの可能性があります。**

1. https://dotnet.microsoft.com/download/dotnet/10.0 を開きます。
2. 「.NET Desktop Runtime」の **x64** インストーラーをダウンロードして実行します。
3. インストール後、LocalCompanion.exe をもう一度起動します。

### 「アプリを開始できません」などのエラーが出る

- Windows 10 1809（ビルド 17763）より古い OS では動作しません。Windows Update を適用してください。
- 展開したフォルダーから `en` `en-us` `en-GB` `ja` `ja-JP` などのフォルダーを削除した場合、起動できないことがあります。ZIP を展開し直してください。

### SmartScreen の「Windows によって PC が保護されました」が出る

本アプリは現在コード署名を行っていないため、初回起動時に表示されることがあります。
「詳細情報」→「実行」を選択すると起動できます。心配な場合は、配布元から入手した ZIP かどうかをご確認ください。

## 初回セットアップが失敗する

### 初回セットアップ画面について

- 初回は **言語選択** のあと、**初回セットアップ** 画面が表示されます。
- **次へ（自動セットアップ）** … llama.cpp の準備と、既定モデル **Gemma 4 E2B（QAT）** の取得を行います。
- **フォルダを選ぶ** … お手持ちの GGUF フォルダを指定すると、既定モデルのダウンロードを省略できます（フォルダは読み取り専用）。

### llama.cpp / モデルのダウンロードに失敗する

- 初回起動時はインターネット接続が必要です。
- 短時間に何度も起動し直すと、ダウンロード元の制限（403 エラー）にかかることがあります。1 時間ほど待ってから再試行してください。
- 会社ネットワークなどでダウンロードがブロックされる場合は、別の回線でセットアップした後、フォルダーごと移動する方法もあります。

### 「DLL が見つかりません」系のエラーが出る

**Visual C++ 再頒布可能パッケージ（x64）が必要です。**

1. https://learn.microsoft.com/cpp/windows/latest-supported-vc-redist を開きます。
2. X64 のインストーラーをダウンロードして実行します。

## チャットができない

### 状態が「OK」にならない

- **⚙ 設定 → モデル** タブの実行状態を確認してください（チャット画面の右下にも同じ状態が表示されます）。
- 他のAIアプリやサーバーアプリが動いている場合は終了してください。改善しない場合は PC を再起動してください。

### 返答が極端に遅い

- CPU のみで動作している可能性があります。設定画面の **モデル** タブで実行バックエンド（CUDA / Vulkan / CPU など）を確認してください。
- メモリ不足の場合は、より小さいモデル（量子化レベルの高い GGUF）をお試しください。**設定 → モデル** で追加モデルフォルダや別の GGUF を選べます。

## モデルが一覧に出ない・切り替わらない

- **設定 → モデル** の「追加モデルフォルダ」で外部フォルダを指定した場合、そのフォルダは **読み取り専用** です。ファイルを移動する必要はありません。
- モデルを変更したあとは **モデルを適用（llama 再起動）** を押してください。
- vision 用 mmproj は、外部フォルダに無い場合 **アプリ側の `models\`** に自動取得されます。

## 読み上げ（VOICEVOX）が動かない

- VOICEVOX は別途インストールが必要です（https://voicevox.hiroshiba.jp/）。
- インストール済みでも、ポート 50021 を他のアプリが使用していると接続できません。
- 標準以外の場所にインストールした場合は、VOICEVOX を先に起動してから LocalCompanion を起動してください。
- 読み上げが動かなくても、チャットなど他の機能は通常どおり使えます。

## 画像入力（Vision）が失敗する

- 本体 GGUF と mmproj ファイルの組み合わせが一致している必要があります。
- 設定画面のモデルタブで、mmproj が設定されているかを確認してください。

## RAG（資料検索）が動かない

- 資料の登録・検索には llama-server の埋め込み API を使用します。状態が「OK」になってからお試しください。
- 1 ファイル最大 20 MB、フォルダー取り込みは最大 200 ファイルまでです。

## ログの場所

問題を報告いただく際は、ログを添えていただくと原因の特定が早くなります。

| 起動方法 | ログ・データの場所 |
|----------|--------------------|
| LocalCompanion.exe | `{LocalCompanion.exe のあるフォルダー}\data\` |

設定画面の「基本」タブにある **ログフォルダーを開く** からも開けます。

## データのバックアップと復元

- 設定画面の「基本」タブにある **データをバックアップ** で、会話・RAG・設定を 1 つの ZIP に書き出せます。
- 復元するときは、アプリを終了した状態で、書き出した ZIP の中身を上記「ログ・データの場所」のフォルダーに上書き展開してください。

## それでも解決しないとき

1. アプリを終了します。
2. LocalCompanion フォルダー内の `tools\llama-cpp\` フォルダーを削除します（次回起動時に再取得されます）。
3. それでも改善しない場合は、設定画面からデータをバックアップしたうえで `data\` フォルダーを別の場所へ移動し、初期状態で起動をお試しください。
