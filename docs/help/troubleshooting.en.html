Troubleshooting

If LocalCompanion does not start or behaves unexpectedly, check this page first.

Will not start

Double-clicking the exe does nothing / the app closes immediately

.NET 10 Desktop Runtime (x64) may not be installed.

  1. Open https://dotnet.microsoft.com/download/dotnet/10.0.
  2. Download and run the x64 .NET Desktop Runtime installer.
  3. Launch LocalCompanion.exe again.

Errors such as “Unable to start the application”

  • Windows 10 version 1809 (build 17763) or later is required. Apply Windows Update if your OS is older.
  • If you removed language folders such as en, en-us, or ja from the extracted folder, the app may fail to start. Extract the ZIP again.

SmartScreen: “Windows protected your PC”

This app is not code-signed yet, so SmartScreen may appear on first launch. Choose More info → Run anyway. Confirm that you obtained the ZIP from a trusted source.

First-time setup fails

About the first-time setup screen

  • On first launch, after language selection, a first-time setup screen appears.
  • Continue with default setup prepares llama.cpp and downloads the default model Gemma 4 E2B (QAT).
  • Choose folder lets you point at your own GGUF folder and skip the default model download (read-only; your files are not modified).

llama.cpp / model download fails

  • First launch requires an internet connection.
  • Restarting the app many times in a short period may hit a download source limit (403). Wait about an hour and try again.
  • If downloads are blocked on a corporate network, set up on another connection and move the folder afterward.

“DLL not found” errors

Visual C++ Redistributable (x64) is required.

  1. Open the Microsoft VC++ redistributable page.
  2. Download and install the X64 package.

Chat does not work

Status never shows OK

  • Check the runtime status on Settings → Model (the same status also appears at the bottom-right of the chat screen).
  • Starting llama-server can take several minutes (especially on first launch or with large models).
  • If another AI app or server app is running, close it. If the problem continues, restart the PC.

Replies are extremely slow

  • The app may be running on CPU only. Check the execution backend (CUDA / Vulkan / CPU, etc.) on the Model tab in Settings.
  • If memory is tight, try a smaller model (higher quantization GGUF). You can choose another GGUF or an additional models folder on the Model tab.

Models missing or will not switch

  • An additional models folder on the Model tab is read-only. You do not need to move files into the app folder.
  • After changing models, press Apply model (restart llama).
  • If mmproj is required for vision, it is downloaded to the app models\ folder when missing from your folder.

Text-to-speech (VOICEVOX) does not work

  • Install VOICEVOX separately. The installer UI is in Japanese; chat works without VOICEVOX.
  • The VOICEVOX tab is hidden until VOICEVOX is installed. Restart LocalCompanion or reopen Settings after installation.
  • On Settings → VOICEVOX, enable speech, choose a speaker, then click Save VOICEVOX settings.
  • Port 50021 must be free for the VOICEVOX engine.
  • If VOICEVOX is installed in a non-default location, start VOICEVOX first, then start LocalCompanion.

Chat behavior

  • Conversations with the default AI (no character selected) are not listed in the left history pane.
  • Under chat Options, turning off Reasoning may stabilize replies.
  • With History off, prior turns are not sent to the model.
  • URL attachments are limited to 2 MB (separate from the 20 MB RAG file limit).

Vision / image input fails

  • The main GGUF and mmproj files must match.
  • On the Model tab in Settings, confirm that mmproj is configured.
  • If Insert image in chat is disabled, mmproj may be missing or the model may not support vision (runtime status “OK” alone does not prove vision works).

RAG (document search) does not work

  • Ingestion and search use the llama-server embedding API. Wait until status is OK.
  • Maximum 20 MB per file; folder import supports up to 200 files.

Log and data locations

You can also use Open log folder on the General tab in Settings.

Backup and restore

  • Use Back up data on the General tab to export conversations, RAG, and settings to a ZIP file.
  • To restore, exit the app and extract the backup ZIP into the data folder listed above.

Still stuck?

  1. Exit the app.
  2. Delete the tools\llama-cpp\ folder inside the LocalCompanion folder (it will be downloaded again on next launch).
  3. If that does not help, back up your data from Settings, move the data\ folder somewhere else, and try a clean start.
